# Guided Scenario Runtime Contract

A scenario runtime card must answer:

1. What story is this?
2. Is it happy path or failure path?
3. What does the user see?
4. What is the first question?
5. What should the learner do next?

A runtime step must answer:

1. What should I read?
2. What should I ask?
3. What evidence should I check?
4. What should I not debug too early?
5. What should I write in my notebook?
6. What is the next action?

This contract keeps future UI simple and guided.
